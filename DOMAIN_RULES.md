# 臺大醫院北護分院長照資源協作平台 - 領域驅動規範 (Domain Rules)

## 1. 地址雙軌處理原則 (Dual-Track Address System)
長照單位的地址經常包含「樓層」、「(總機)」甚至「分機號碼」，這會導致 Google Maps 導航完全失效。
- **UI 顯示 (`address`)**：必須保留最原始、最完整的地址與樓層資訊，因為個管師與社工需要知道具體位於幾樓之幾。
- **導航專用 (`navAddress`)**：必須在 ETL 階段透過 Regex 剔除括號備註、樓層與「之X」，自動補齊缺少的新北市/臺北市，產出純粹的街廓地址供 Google API 使用。

## 2. 實務轉介欄位保留原則 (Referral Method Preservation)
轉介方式是派案的核心。文字簡化必須極度小心：
- **實體聯絡資訊絕對保留**：若文字中包含 `LINE`, `http`, `www`, `電話：`, `@`，代表這是實質的聯絡管道，絕對不可被過度簡化覆寫。
- **語意分流**：將單純的「提供各單位轉介」統一標準化為 `「需單位轉介」`；將「線上表單」、「民眾自行至全聯」統一標準化為 `「可自行申請」` 並保留原有表單名稱，以利於前端快速辨識。

## 3. 無實體地址資源處理 (Non-Physical Resources)
- 「交通接送」與部分「經濟安全」服務是車隊或補助，不具備實體據點。
- **清單保留**：在左側的資料清單中必須完整保留這些資源，不可刪除。
- **地圖過濾**：在傳入 Maplibre 繪製標記前，必須實作前端過濾，阻止這類資料被繪製在 `(0,0)`，並且在地圖卡片中必須隱藏導航按鈕。
